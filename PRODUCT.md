# Product

## Register

product

## Users

Local Suite is for one local developer managing many active repositories, Docker stacks, ports, git states, and safe local actions on a single workstation.

## Product Purpose

Local Suite exists to answer what is running, what needs attention, what is exposed, and what can be safely inspected next. Success means the user can scan the fleet quickly without secrets, filler UI, destructive actions, or noisy local state hiding real exceptions.

## Brand Personality

Calm, exact, operational.

## Anti-references

Avoid marketing dashboards, decorative metric cards, card spam, nested panels, oversized chrome, vague helper copy, filler controls, one-note gray UI, and layouts that try to show every possible detail at once.

## Design Principles

- Exceptions first: public exposure, unresolved listeners, dirty repos, heavy stacks, and failed checks outrank general metrics.
- One control surface: each region needs one job and one obvious next action.
- Dense, not cluttered: tables and compact rows are preferred over repeated cards when the data is comparable.
- No fake safety: dry-run, read-only, preview, and approval boundaries must be visible near the action.
- Short copy only: labels should help the user decide or act.

## Accessibility & Inclusion

Target WCAG AA contrast, visible focus states, keyboard-reachable controls, reduced-motion-safe interactions, and layouts that remain usable at 390px mobile width without horizontal page overflow.
